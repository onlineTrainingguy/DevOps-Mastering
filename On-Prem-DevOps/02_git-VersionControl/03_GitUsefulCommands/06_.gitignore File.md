# `.gitignore` Explained

`.gitignore` is a special file in Git that tells Git which files and directories should be **ignored** and not tracked.

---

# Why Use `.gitignore`?

Many files should not be stored in Git, such as:

* Temporary files
* Log files
* Build artifacts
* Dependency directories
* IDE settings
* Secrets and environment files

Examples:

* `*.log`
* `node_modules/`
* `bin/`
* `obj/`
* `.env`
* `.vscode/`

---

# Basic Example

Create a `.gitignore` file:

```bash
touch .gitignore
```

Add patterns:

```gitignore
*.log
temp/
node_modules/
.env
```

---

# Pattern Meanings

| Pattern  | Meaning                      |
| -------- | ---------------------------- |
| `*.log`  | Ignore all `.log` files      |
| `temp/`  | Ignore the `temp` directory  |
| `.env`   | Ignore the `.env` file       |
| `*.tmp`  | Ignore all `.tmp` files      |
| `build/` | Ignore the `build` directory |

---

# Example Directory Structure

```text
project/
├── app.py
├── debug.log
├── .env
├── temp/
│   └── test.txt
└── .gitignore
```

With this `.gitignore`:

```gitignore
*.log
.env
temp/
```

Git tracks only:

* `app.py`
* `.gitignore`

---

# Ignore Files by Extension

```gitignore
*.txt
*.log
*.bak
```

---

# Ignore Specific Files

```gitignore
config.json
secret.key
```

---

# Ignore Directories

```gitignore
node_modules/
dist/
build/
```

---

# Ignore Files Recursively

```gitignore
**/*.tmp
```

Ignore all `.tmp` files in all subdirectories.

---

# Negation (`!`)

Track a specific file even if a broader pattern ignores it.

```gitignore
*.log
!important.log
```

All `.log` files are ignored except `important.log`.

---

# Comments

```gitignore
# Ignore log files
*.log
```

---

# Important Note

`.gitignore` affects only **untracked files**.

If a file is already tracked, adding it to `.gitignore` does not stop Git from tracking it.

---

# Check Ignored Files

```bash
git status --ignored
```

# Direct Summary

> `.gitignore` tells Git which files and folders should be ignored and not added to the repository. It is commonly used to exclude temporary files, build outputs, IDE settings, and sensitive files such as `.env` or secrets.
